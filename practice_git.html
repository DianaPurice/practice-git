Git Cheat sheet

Always pull the main repusotory before making any changes.

Create issues and new branches for each task.

Merge your branches once you are sure the task has been completed and then close the issue for the task.

Sepparate issues based on the level of the page you are working on.

Add and commit the changes before you push your code.

Push the changes once you have completed a task.

Pulling repositories

Git command Explanation
git pull origin
[repository-name]		 Pulls the GitHub code from the named repository

Pushing repositories

Git command Explanation
git push origin
[repository-name]		 Pushes the local code to the named repository on GitHub

Cloning repositories

Git command Explanation
git clone [SSH from GitHub] Creates a local copy of the GitHub repostory

Version controls

Git command Explanation
git checkout [branch name] Switches to an existing branch
git checkout -b [branch name] Switches to a new branch

Repositories issues

Step Explanation
Open issue Create issues to be able to track them easier
Issue element
Title The name of the issue
Description User story
Close issue Once everything is done close the issue
